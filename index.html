topic 1. css 등장 이전의 상황 html에 대한 아름다움의 불만들이 커져가면서 css라는 새로운 아름다움에 대한 언어가 나타났다. html을 쓸 때 정보에 대한 태그와 정보가 아닌 태그를 구분하여 작성을 해야한다(정보 자체의 질을 높이기 위함) 극한의 상황으로 생각하여 css라는 것은 html을 더 편리하고, 보기 좋고, 효율적으로 만들어 줄 것이다.

topic 2. css의 등장(!-- 콘텐츠 --)는 괄호에 포함되어 있는 콘텐츠를 무시하는 코드! (style)(/style)은 html의 문법이면서 style태그 안쪽의 내용은 css로 읽는다! 라는뜻임 css는 중복을 막는 효과를 가진다( 같은 태그를 고쳐야 하는 경우에 선택자,선언,property,value를 변경하면 모두 변경된다. 유지보수성,가독성이 높아진다. 정보 자체의 질도 높아진다. html에서 디자인에 대한 기능을 가져온 것이 css이며, css를 통해서 웹페이지를 디자인하는 것이 html을 이용하는 것보다 더 효율적이기 때문에 css를 사용한다.

topic3. css의 기본 문법 style의 속성은 사용하면 웹브라우저는 css의 문법에따라 해석된 결과를 style 속성이 위치하고 있는 태그에 적용한다. 선택자(selector): 웹페이지에서 주고 싶은 효과를 누구에게 줄 지 선택한다. ex) a, li, h1.. 선언,효과(declaration): 웹페이지에 어떤 효과를 줄지를 선택한다 ex) color, border. 웹페이지에서 css를 사용하는 2가지 방법은 첫째는 스타일 태그를 사용하는 것, 둘째는 스타일 속성을 사용하는 것(이 경우 스타일 속성 특징에 따라 선택자는 필요없다. why? 위치하고있는 곳에 적용하니까!) 글자를 꾸미는 속성은 text-decoration; 세미콜론은 시작과 끝을 알려주는 구분자로써 사용되고 value까지 작성을 다 하면 세미콜론을 꼭 붙여준다.﻿
